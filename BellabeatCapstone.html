Process

Given the limited time span of the dataset, only data available across both periods will be merged and utilized for daily analysis to ensure consistency and reliability. To maintain the integrity of the sample, datasets with fewer than 20 participants will be excluded, given the small overall sample size. However, the complete hourly step and hourly sleep data will be retained to analyze user behavior patterns on an hourly basis.

SQL in BigQuery will be used to process the data. As an industry-standard language, SQL is ideal for managing structured datasets, and BigQuery enhances its capabilities with high-speed processing, scalability, and advanced analytics features. This combination ensures efficient data cleaning, transformation, and analysis, making it well-suited for identifying trends and patterns in large datasets.

Step 1: Change Time Format

Datasets containing minute/second-level data are formatted as MM/DD/YYYY HH:MM:SS AM/PM, which is incompatible with the SQL DATETIME format YYYY-MM-DD HH:MM:SS. To address this without losing data, the format was converted using Python. The following script demonstrates how this was achieved:

import pandas as pd

# Read the dataset

df = pd.read_csv('file.csv')

# Convert the 'Time' column to SQL-compatible DATETIME format

df['Time'] = pd.to_datetime(df['Time'], format='%m/%d/%Y %I:%M:%S %p')

# Save the formatted dataset to a new CSV file

df.to_csv('formatted_file.csv', index=False)

This process ensures that the data aligns with SQL requirements, facilitating seamless integration and further analysis.

Daily Activity Table

The final daily activity table was created by following these steps:

  1. Combine and round up the average minute METs by date.

The average METs per minute were calculated and rounded to two decimal places for each user by date using the query below:

SELECT Id, DATE(ActivityMinute) ActivityDate, ROUND(AVG(METs),2) AvgMETs

FROM combined_table

GROUP BY Id, DATE(ActivityMinute)

  1. Combine and round up the sum of sleep hours by date.

The total sleep hours, as well as asleep and awake percentages by date, were calculated using the following query:

SELECT Id, DATE(`date`) ActicityDate,ROUND(COUNT(*)/60,2) TotalHour,ROUND(COUNT(CASE WHEN value = 1 THEN 1 END)/COUNT(*),2) AsleepPercent,

ROUND(COUNT(CASE WHEN value = 3 THEN 3 END)/COUNT(*),2) AwakePercent

FROM combined_table

GROUP BY Id, DATE(`date`)

  1. Filter and merge daily activity data.

The daily activity data required additional cleaning to resolve duplicates and inconsistencies for the date 2016-04-12, as records were present in both periods. When duplicate data was combined, records for 14 users showed more than 1,440 minutes (24 hours) of activity for that date. Furthermore, some records contained exactly 1,440 sedentary minutes, while others showed varied active minutes. These anomalies were addressed by:

  1. Removing records with 1,440 sedentary minutes (the sample size remained unchanged after removal).
  2. For remaining duplicates, keeping the record with the highest total minutes, calculated as:

TotalMinutes = VeryActiveMinutes + FairlyActiveMinutes + LightlyActiveMinutes + SedentaryMinutes

The cleaning was implemented using the following queries:

WITH filtered_data AS(

ranked_data AS (

FROM ranked_data

User Profile

Users are categorized based on the recommended daily sleep hours and weekly active hours to better understand their behavior patterns. Given the relatively small sample size, individual data will be examined to provide deeper insights.

Sleep Patterns

On average, the sample population sleeps 7.15 hours per day. Users are categorized into three groups based on their average daily sleep duration:

  • Short Sleepers: Less than 7 hours per night
  • Normal Sleepers: 7–9 hours per night (recommended range)

This classification is aligned with the National Heart, Lung, and Blood Institute (NHLBI) and Johns Hopkins Medicine recommendation that healthy adults sleep 7–9 hours per night.

The analysis shows that:

Data Correlations

Daily steps are commonly used as a key benchmark for health tracking. This analysis explores correlations between steps and other attributes, including calories, sleep, and average METs per minute. Additionally, the analysis examines the relationship between activity levels, categorized by intensity, and other attributes to provide a more detailed understanding of these correlations.

Before delving into the analysis, it is important to clarify what a MET represents. A MET (Metabolic Equivalent of Task) is a ratio of a person's working metabolic rate to their resting metabolic rate. One MET corresponds to the energy expended while sitting at rest, also known as the basal metabolic rate. For example, an activity with a MET value of 4 indicates that the person is using four times the energy compared to resting.

In this dataset, MET values have been multiplied by 10 to enhance trend visibility and facilitate analysis.
